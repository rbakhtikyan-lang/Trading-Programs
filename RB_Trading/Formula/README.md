# 📊 Программа для Трейдинга

Программа для подключения к криптовалютным биржам (CEX), загрузки исторических данных свечей и выполнения математического анализа на основе данных High/Low.

## 🚀 Возможности

1. **Подключение к CEX** - Поддержка различных криптовалютных бирж через библиотеку CCXT
2. **Загрузка данных** - Получение исторических данных свечей (OHLCV) для любой монеты
3. **Сохранение в БД** - Хранение данных High/Low с датами в PostgreSQL
4. **Математический анализ** - Вычисление различных метрик и статистики
5. **Красивый вывод** - Форматированный вывод в терминале с цветовым кодированием

## 🔧 Установка

### 1. Установка зависимостей Python

```bash
pip install -r requirements.txt
```

### 2. Установка PostgreSQL

Ubuntu/Debian:
```bash
sudo apt update
sudo apt install postgresql postgresql-contrib
```

macOS:
```bash
brew install postgresql
```

### 3. Создание базы данных

```bash
# Войти в PostgreSQL
sudo -u postgres psql

# Создать базу данных
CREATE DATABASE trading_db;

# Создать пользователя (опционально)
CREATE USER trading_user WITH PASSWORD 'your_password';
GRANT ALL PRIVILEGES ON DATABASE trading_db TO trading_user;

# Выйти
\q

# Выполнить SQL скрипт
sudo -u postgres psql -d trading_db -f base.sql
```

### 4. Настройка подключения

Отредактируйте параметры подключения к БД в файле `main.py`:

```python
self.conn = psycopg2.connect(
    dbname="trading_db",
    user="postgres",          # Ваш пользователь
    password="your_password",  # Ваш пароль
    host="localhost",
    port="5432"
)
```

## 🎯 Использование

### Запуск программы

```bash
python main.py
```

### Главное меню

После запуска вы увидите меню с опциями:

```
1. Подключиться к CEX
2. Загрузить свечные данные
3. Показать сохраненные данные
4. Выполнить анализ (RB Formula)
5. Статистика по монете
0. Выход
```

### Пример использования

1. **Подключение к бирже**
   - Выберите опцию `1`
   - Нажмите Enter для demo режима (без API ключей)
   - Или введите свои API ключи

2. **Загрузка данных**
   - Выберите опцию `2`
   - Введите символ: `BTC/USDT`
   - Введите таймфрейм: `1h`
   - Введите количество свечей: `100`
   - Сохраните данные в БД: `y`

3. **Просмотр данных**
   - Выберите опцию `3`
   - Введите символ или нажмите Enter для всех данных

4. **Анализ RB Formula**
   - Выберите опцию `4`
   - Введите символ: `BTC/USDT`
   - Введите таймфрейм: `1h`

## 📊 Что анализирует RB Formula

### High/Low Анализ
- Максимальные и минимальные значения High/Low
- Средние значения
- Общий диапазон цен

### Анализ диапазонов свечей
- Средний, максимальный, минимальный диапазон
- Медианный диапазон
- Стандартное отклонение

### Движение цены
- Общее и среднее движение
- Процент бычьих/медвежьих свечей
- Количество нейтральных свечей

### Анализ объема
- Общий, средний, максимальный объем
- Медианный объем

### Волатильность
- ATR (Average True Range)
- Стандартное отклонение
- Коэффициент вариации
- Уровень волатильности

### Поддержка и Сопротивление
- Определение уровней поддержки
- Определение уровней сопротивления
- Текущий ценовой диапазон

## 🎨 Цветовое кодирование

- 🟢 **Зеленый** - Положительные значения, рост, максимумы
- 🔴 **Красный** - Отрицательные значения, падение, минимумы
- 🔵 **Синий** - Нейтральные значения, средние показатели
- 🟡 **Желтый** - Заголовки, предупреждения
- 🟣 **Фиолетовый** - Даты, временные метки
- ⚪ **Белый** - Описания, метки

## 📝 Структура базы данных

### Таблица `candles`
Хранит данные свечей:
- symbol - торговая пара
- timeframe - таймфрейм
- timestamp - время в миллисекундах
- datetime - дата и время
- high, low, open, close - цены
- volume - объем торгов

### Таблица `analysis_results`
Хранит результаты анализа

### Таблица `support_resistance_levels`
Хранит уровни поддержки/сопротивления

### Таблица `event_log`
Логирование событий системы

## ⚙️ API ключи

Для получения публичных данных (история свечей) API ключи **не требуются**.

Для приватных операций (торговля, баланс):
1. Зарегистрируйтесь на бирже
2. Создайте API ключи в настройках
3. Введите их при подключении к CEX

## 🛠️ Дополнительные функции

### Загрузка всей истории

В модуле `cex_api.py` есть функция `fetch_all_historical_data()` для загрузки всех доступных данных:

```python
connector = CEXConnector()
all_candles = connector.fetch_all_historical_data('BTC/USDT', '1d')
```

### SQL функции

В БД доступны полезные функции:
- `count_candles(symbol, timeframe)` - подсчет свечей
- `cleanup_old_candles(days)` - очистка старых данных

### Views (представления)

- `latest_candles` - последние свечи по каждому символу
- `symbol_statistics` - статистика по символам

## 🐛 Устранение неполадок

### Ошибка подключения к БД
```
FATAL: password authentication failed
```
Проверьте пароль в `main.py` и права пользователя PostgreSQL.

### Ошибка подключения к CEX
```
NetworkError
```
Проверьте интернет-соединение и доступность биржи.

### Символ не найден
```
Symbol not found
```
Используйте правильный формат: `BTC/USDT`, `ETH/USDT` и т.д.
